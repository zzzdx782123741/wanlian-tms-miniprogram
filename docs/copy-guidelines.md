# 万联驿站小程序中文文案基准表

本文件用于统一小程序前端的中文文案口径，减少同一业务在不同页面出现多套说法。

适用范围：
- `pages/`
- `components/`
- `custom-tab-bar/`
- `utils/` 中直接面向页面展示的文案

基准原则：
- 优先使用短句，避免口语化和重复修饰。
- 同一状态只保留一种主文案，不在不同页面写成近义词。
- 按钮文案优先使用“动作 + 对象”或“立即 + 动作”。
- 空状态、错误提示、成功提示分别保持统一句式。
- 订单状态文案以 `utils/order-config.js` 为唯一基准来源。

## 1. 订单状态基准

| 状态值 | 标准文案 | 备注 |
| --- | --- | --- |
| `awaiting_fleet_approval` | 待车队审批 | 车队审批前 |
| `awaiting_time_confirmation` | 待确认到店时间 | 不使用“待确认时间” |
| `pending_assessment` | 待接车检查 | 技师开始检查前 |
| `awaiting_approval` | 待审批报价 | 不使用“报价待审批” |
| `in_repair` | 维修中 | 保养场景可按页面语义显示“服务进行中”，但列表页优先统一为“处理中”标签 |
| `awaiting_addon_approval` | 待审批增项 | 不使用“增项待审批” |
| `pending_confirmation` | 待确认完工 | 不使用“待确认” |
| `completed` | 已完成 | 终态 |
| `rejected` | 已拒绝 | 终态 |
| `refunded` | 已退款 | 终态 |

补充约定：
- 列表页标签统一使用：`全部 / 待处理 / 处理中 / 已完成`
- 页面摘要或待办分组可用：`待审批增项 / 待确认完工`
- 技师详情中的动作提示可写“请先确认到店时间，再开始接车检查”

## 4. 空状态文案

统一模式：
- 主文案：说明“暂无什么”
- 辅助文案：说明下一步操作或当前原因

推荐写法：

| 场景 | 主文案 | 辅助文案 |
| --- | --- | --- |
| 通用订单列表 | 暂无订单记录 | 当前没有可展示的订单 |
| 车辆筛选后的订单列表 | 该车辆暂无订单 | 可切换标签查看该车辆的其他订单，或清除筛选 |
| 门店订单列表 | 暂无订单 | 当前筛选条件下暂无订单 |
| 我的车辆 | 暂无车辆记录 | 点击上方按钮新增车辆 |
| 车队车辆 | 暂无车辆记录 | 请先将车辆添加到车队 |
| 地址列表 | 暂无服务地址 | 点击下方按钮新增地址 |
| 商品选择 | 暂无商品 | 请尝试更换搜索关键词或分类 |
| 在途订单 | 暂无在途订单 | 仅在有运输任务时显示 |

## 5. 加载、成功、失败提示

统一模式：
- 加载中：`加载中...`
- 提交中：`提交中...`
- 处理中：`处理中...`

失败提示优先写明对象，不使用过于泛化的“失败”：

| 场景 | 标准文案 |
| --- | --- |
| 加载订单 | 加载订单失败 |
| 加载车辆 | 加载车辆失败 |
| 加载商品 | 加载商品失败 |
| 加载进度 | 加载进度失败 |
| 加载财务数据 | 加载财务数据失败 |
| 加载报表 | 加载报表失败 |
| 删除 | 删除失败 |

## 6. 注册与入驻相关文案

统一约定：
- 使用“入驻申请”，不混用“注册申请”
- 使用“入驻协议”，不混用“注册协议”
- 使用“审核通过后，将使用该账号登录系统”

推荐写法：
- `请先阅读并同意入驻协议`
- `车队入驻申请已提交，请等待平台审核。审核通过后，我们将使用管理员账号信息联系您。`
- `门店入驻申请已提交，请等待平台审核。审核通过后，我们将使用管理员账号信息联系您。`

## 7. 维护要求

- 新增订单状态文案时，先更新 `utils/order-config.js`
- 页面内若需要局部变体，应在代码注释中说明原因，避免后续又改回另一套说法
- 新页面上线前，优先检查：
  - 状态词是否复用基准表
  - 按钮是否出现“去审批 / 去确认 / 去评价”这类旧文案
  - 空状态是否包含主文案和辅助文案
  - 错误提示是否明确指出失败对象
